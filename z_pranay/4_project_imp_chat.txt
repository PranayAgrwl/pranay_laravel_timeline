tree-style foldering and compartmentalising  (apply this EVERYWHERE)

    each domain area lives in its own folder, not flat soup. think
    in terms of small, single-purpose folders that read like a table
    of contents.

    inside a module (Modules/<Name>/):
        app/
            Http/Controllers/        <- one controller per area, thin
            Livewire/                <- one sub-folder per area
                <Area>/
                    Index.php
                    ... more if needed
            Models/                  <- domain entities
                <Entity>.php
                Labels/              <- lookup-only models in their own subfolder
                    <Lookup>.php
            Providers/
            Listeners/               <- model events (auto-mirror, etc.)
            Mail/                    <- if the area sends email
        database/
            migrations/              <- dependency-ordered, one table per file
            seeders/                 <- per-module seed (only if explicitly wanted)
        resources/
            views/
                components/layouts/  <- module base layout
                partials/            <- shared partial includes (navbar, etc.)
                <area>/              <- one folder per area
                livewire/<area>/     <- partials for the area's livewire
        routes/
            web.php / api.php

    each area folder is self-contained; no cross-area peeking.
    if a thing is shared between areas, it belongs at the module root,
    NOT inside one of the areas.

    naming:
        - migration file names dependency-ordered via timestamp
        - controllers + models singular nouns; tables plural-prefixed
        - lookup tables grouped under Models/Labels/ in the model layer
